# c0dx: Telegram Notify Enhanced

WordPress-плагин для отправки уведомлений о заказах WooCommerce и заявках Contact Form 7 в Telegram.

Текущая версия плагина: **2.3.0**.

Плагин поддерживает нескольких подписанных получателей, настраиваемые шаблоны сообщений, HTML и MarkdownV2, фоновую очередь, повторные попытки, тестовые отправки и компактную диагностику.

## Состояние проекта

Версия 2.3.0 — текущая рабочая публичная база. Интерфейс получателей, сохранение настроек и тестовая отправка Contact Form 7 проверены на практике.

Уведомления WooCommerce и защита от дублей реализованы, но сложные сценарии с повторными переходами заказа между статусами рекомендуется отдельно проверить под бизнес-процесс конкретного магазина.

Дальнейшие изменения вносятся по мере появления практической необходимости, результатов тестирования и требований совместимости.

## Возможности

- Уведомления о новых заказах WooCommerce и изменениях статусов.
- Уведомления об отправках Contact Form 7.
- Несколько Telegram-чатов, групп или каналов с подписями только для админки.
- Массовый импорт получателей с необязательными подписями в формате `ID | Имя`.
- Числовые Chat ID и публичные каналы вида `@channelusername`.
- Раздельные шаблоны сообщений для WooCommerce и Contact Form 7.
- Режимы форматирования HTML и MarkdownV2.
- Фоновая очередь с ограниченным числом повторных попыток.
- Повторная отправка только в неудачные чаты после частичной ошибки.
- Тестовые отправки заказа, Contact Form 7 и произвольного сообщения.
- Компактная диагностика очереди и ошибок.
- Совместимость с WooCommerce HPOS через CRUD API заказов.
- Возможность задать Bot Token и список получателей через `wp-config.php`.
- Без Composer- и npm-зависимостей.

## Требования

- WordPress 6.8+
- PHP 8.0+
- WooCommerce 6.0+ — для уведомлений о заказах
- Contact Form 7 — для уведомлений о формах
- исходящий HTTPS-доступ к `https://api.telegram.org` через WordPress HTTP API

Интеграции WooCommerce и Contact Form 7 включаются и отключаются независимо. Для работы плагина не требуется одновременная установка обоих расширений.

## Проверенная конфигурация

Текущая версия вручную проверена на:

- WordPress 7.0;
- PHP 8.4;
- MySQL 5.7;
- тестовой отправке Contact Form 7;
- успешных и ошибочных Telegram-получателях;
- подписанных получателях и массовом импорте.

Другие поддерживаемые версии WordPress, PHP, базы данных и интеграций могут работать, но не все сочетания проверены во всех сценариях доставки.

## Установка

### Установочный ZIP из GitHub Release

Используйте файл `tgnotify.zip`, приложенный к GitHub Release, когда релиз опубликован. Автоматические архивы GitHub **Source code** являются снимками всего репозитория и не предназначены для прямой загрузки через установщик WordPress.

### Ручная установка из репозитория

1. Скопируйте папку `tgnotify` в `/wp-content/plugins/`.
2. Активируйте **c0dx: Telegram Notify Enhanced**.
3. Откройте **Telegram Notify** в основном меню WordPress или нажмите **Настройки** в списке плагинов.
4. Укажите Telegram Bot Token и добавьте одного или нескольких получателей.
5. Настройте шаблоны, режим форматирования и нужные типы уведомлений.
6. Выполните тестовые отправки до включения уведомлений на рабочем сайте.

## Telegram Bot Token

Создать Telegram-бота и получить токен можно через [@BotFather](https://t.me/BotFather).

Токен можно хранить вне базы WordPress:

```php
define( 'TN_TELEGRAM_BOT_TOKEN', '123456:ABC-DEF...' );
```

Если определена константа `TN_TELEGRAM_BOT_TOKEN`, токен не отображается на странице настроек и не перезаписывается при сохранении формы.

Никогда не публикуйте реальный Bot Token в репозитории, issue, скриншоте или журнале ошибок.

## Получатели и Chat ID

Каждый получатель состоит из двух полей:

- **Chat ID** — идентификатор личного чата, группы или канала;
- **Описание** — необязательная подпись, используемая только в админке.

Примеры:

```text
200012345
-1002123456789
@channelusername
```

Числовые идентификаторы групп и каналов обычно отрицательные и часто начинаются с `-100`.

Массовый импорт принимает переносы строк, запятые и точки с запятой. Подпись можно добавить в формате:

```text
200012345 | Иван
-1002123456789 | Рабочая группа
@channelusername | Публичный канал
```

Дублирующиеся получатели повторно не добавляются.

Получателей также можно задать в `wp-config.php`:

```php
define( 'TN_TELEGRAM_CHAT_IDS', '-1002123456789,@channelusername' );
```

Если определена `TN_TELEGRAM_CHAT_IDS`, этот список имеет приоритет над получателями из базы. Сохранённые подписи остаются доступны в административном интерфейсе.

## Фоновая очередь

При включённой фоновой отправке сообщение временно сохраняется и обрабатывается отдельной задачей WP-Cron.

- Для одного задания выполняется до трёх попыток.
- Интервал между попытками постепенно увеличивается.
- После частичной ошибки в очередь возвращаются только неудачные Chat ID.
- Уведомление о заказе считается обработанным только после подтверждённой отправки.
- Просроченные задания удаляются автоматически.
- Очередь можно очистить вручную на странице настроек.

WP-Cron зависит от посещаемости сайта. На малопосещаемых проектах возможна задержка, если запуск WordPress cron не настроен на сервере.

## Форматирование

Плагин поддерживает Telegram HTML и MarkdownV2.

Динамические данные заказа и формы экранируются автоматически. Статический текст MarkdownV2, введённый администратором, должен соответствовать правилам экранирования Telegram.

## Диагностика и конфиденциальность

- На странице настроек отображается краткий последний результат или ошибка.
- Временная диагностика фиксирует работу очереди и события доставки.
- Полные успешные ответы Telegram API не сохраняются.
- Текст сообщения в фоновой очереди может временно содержать данные клиента или формы.
- Очередь хранится не более суток, отдельное задание — не более шести часов.
- Данные отправляются только в Telegram API в рамках заявленного назначения плагина.

Проверьте, какие поля включены в шаблоны уведомлений, и применяйте требования к персональным данным, действующие для сайта и его юрисдикции.

## Отдельная область тестирования

Защита от повторных уведомлений о статусах WooCommerce намеренно оставлена консервативной. Перед рабочим использованием рекомендуется проверить:

- создание нового заказа;
- автоматический переход `pending → processing` после оплаты;
- ручные смены статуса;
- возврат заказа в ранее использованный статус;
- работу с включённой и выключенной фоновой очередью;
- восстановление после временной ошибки Telegram API.
