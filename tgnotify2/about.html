c0dx: Telegram Notify Enhanced

Версия 2.2.0.

Плагин отправляет уведомления о заказах WooCommerce и сообщениях Contact Form 7 в один или несколько Telegram-чатов или каналов.

Уведомления WooCommerce и Contact Form 7 можно включать отдельно. Для CF7 также можно ограничить отправку выбранными формами.

Основные возможности

  • Уведомления о новых заказах и смене статусов WooCommerce.
  • Уведомления об отправках Contact Form 7.
  • Несколько chat ID или каналов вида @channelusername.
  • Шаблоны сообщений с плейсхолдерами.
  • Форматирование HTML или MarkdownV2.
  • Фоновая очередь с повторными попытками.
  • Тестовые отправки из настроек плагина.
  • Поддержка WooCommerce HPOS через CRUD API заказов.

Требования и целевая совместимость

  • WordPress 6.8–7.0+.
  • PHP 8.0+.
  • WooCommerce 6.0+ — для уведомлений о заказах.
  • Contact Form 7 — для уведомлений о формах.
  • Исходящий HTTPS-доступ к api.telegram.org через WordPress HTTP API.

Перед обновлением рабочего сайта рекомендуется выполнить проверку на тестовой копии.

Bot Token

  1. Откройте в Telegram бота @BotFather.
  2. Создайте нового бота командой /newbot.
  3. Скопируйте выданный токен и вставьте его в поле Bot Token в настройках плагина.

Токен можно хранить вне базы WordPress, добавив в wp-config.php:

define( 'TN_TELEGRAM_BOT_TOKEN', '123456:ABC-DEF...' );

Если константа определена, токен не выводится в интерфейсе и не перезаписывается при сохранении настроек.

Chat ID

  1. Отправьте любое сообщение своему боту или в нужный канал.
  2. Для канала предварительно добавьте бота в администраторы.
  3. Получите числовой chat ID через Telegram API или используйте имя публичного канала вида @channelusername.

Можно указать несколько значений, разделяя их запятой или новой строкой.

Chat ID также можно задать в wp-config.php:

define( 'TN_TELEGRAM_CHAT_IDS', '-1001234567890,@channelusername' );

Асинхронная отправка

При включённой фоновой отправке сообщение сначала сохраняется в очереди WordPress и затем обрабатывается отдельным заданием WP-Cron.

  • Для одной задачи выполняется не более трёх попыток.
  • Интервал между попытками постепенно увеличивается.
  • При частичной ошибке повторная отправка выполняется только в неудачные чаты.
  • Заказ помечается обработанным только после подтверждённой отправки.
  • Просроченные задания автоматически удаляются.

Важно: WP-Cron зависит от посещений сайта. На малопосещаемом сайте фоновая отправка может выполняться с задержкой.

Очередь и диагностика

  • Очистка очереди: удаляет зависшие фоновые задания.
  • Последняя ошибка: хранит краткое описание последней проблемы отправки.
  • Отладочные логи: показывают работу очереди и диагностические сообщения.
  • Полный успешный JSON-ответ Telegram API в базе не сохраняется.

Шаблоны HTML

Поддерживаются основные теги Telegram:

  • <b>, <strong> — жирный текст;
  • <i>, <em> — курсив;
  • <u> — подчёркивание;
  • <code>, <pre> — моноширинный текст и блок кода;
  • <a href="..."> — ссылка;
  • <br> — перенос строки.

Пример:

<b>Заказ #{order_id}</b>
Статус: {from_status} → {to_status}
Клиент: {customer_name}
Телефон: {phone}
Email: {email}
Сумма: {total}
Товары:
{items}
<a href="{order_link}">Открыть заказ</a>

Шаблоны MarkdownV2

Динамические данные заказа и формы экранируются автоматически. Специальные символы в статичном тексте пользовательского шаблона необходимо экранировать самостоятельно.

К специальным символам относятся:

_ * [ ] ( ) ~ ` > # + - = | { } . !

Пример:

*Заказ #{order_id}*
Статус: {from_status} → {to_status}
Клиент: {customer_name}
Телефон: {phone}
Email: {email}
Сумма: {total}
Товары:
{items}
[Открыть заказ]({order_link})

Плейсхолдеры WooCommerce

  • {order_id} — номер заказа;
  • {from_status} — предыдущий статус;
  • {to_status} — новый статус;
  • {customer_name} — имя покупателя;
  • {phone} — телефон;
  • {email} — email;
  • {total} — итоговая сумма;
  • {items} — список товаров;
  • {order_link} — ссылка на заказ в админке.

Плейсхолдеры Contact Form 7

  • {form_title} — название формы;
  • {all_fields} — все пользовательские поля формы;
  • {field_{field_name}} — конкретное поле формы.

Технические поля, имена которых начинаются с символа _, в сообщение не включаются.

Конфиденциальность

При фоновой отправке текст сообщения временно сохраняется в WordPress transient. Он может содержать имя, телефон, email, сведения о заказе и другие поля формы.

  • Очередь хранится не более суток.
  • Отдельное задание считается просроченным через шесть часов.
  • Данные отправляются только в настроенные Telegram-чаты через Telegram API.

Ограничения

  • Не публикуйте Bot Token в коде, документации или открытом репозитории.
  • Максимальная длина одного сообщения Telegram — 4096 символов.
  • Большие шаблоны и формы могут потребовать сокращения содержимого.
  • Логика защиты от повторных уведомлений о статусах требует дополнительного тестирования при многократном возврате заказа к одному и тому же статусу.
