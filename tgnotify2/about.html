Telegram Notify — краткое описание и инструкция

Плагин позволяет отправлять уведомления о новых заказах в WooCommerce и сообщениях, отправленных через формы плагина Contact Form 7.

Уведомления можно отдельно включить/выключить для WooCommerce и Contact Form 7, а также отдельно для определенных форм.

Когда пригодится

  • Магазину — мгновенно получать статусы новых заказов и изменения оплаты.
  • Администратору или менеджеру — оперативно реагировать на заявки с сайта и корректировать статусы заказов.
  • Команде поддержки — перенаправлять клиентские сообщения и отслеживать формы обратной связи.

Асинхронная отправка

Опция «Не блокировать выполнение запроса (отправка через фон)» складывает сообщения в очередь и отправляет их отдельным фоновым процессом. Это позволяет не задерживать загрузку страниц сайта, даже если Telegram API отвечает с задержкой.

Bot Token

  1. Откройте в Telegram бота @BotFather и отправьте команды /start, затем /newbot.
  2. Введите желаемое имя и уникальное username (должен заканчиваться на Bot, например MyShopBot).
  3. Скопируйте выданный BotFather токен и вставьте его в поле Bot Token в настройках плагина.

Минимальные требования

  • WordPress 6.0+, WooCommerce 6.0+ (при использовании заказов).
  • PHP 8.0+ с поддержкой HTTPS и расширения cURL (или WordPress HTTP API).
  • Возможность исходящих HTTPS-запросов к api.telegram.org.

Chat ID(s)

  1. Отправьте любое сообщение своему боту или в нужный канал (для каналов предварительно добавьте бота в администраторы).
  2. Перейдите по адресу https://api.telegram.org/bot<ВАШ_ТОКЕН>/getUpdates и найдите в ответе JSON объект chat с полем id.
  3. Альтернатива: напишите команду /start боту @userinfobot — он пришлёт ваш chat id мгновенно.

Для публичного канала можно использовать формат @channelusername или числовой chat id.

Очередь и логи

  • Очистка очереди: на странице настроек нажмите «Очистить очередь сообщений», чтобы удалить зависшие задания.
  • Повторные попытки: при фоновой отправке неудачные сообщения автоматически возвращаются в очередь с ограниченным числом повторов.
  • Отладка: блок «Последняя ошибка» и «Отладочные логи» в настройках помогает увидеть ответы Telegram API и диагностировать проблемы.

Памятка по шаблонам

HTML (Parse Mode)

  • <b> / <strong> — жирный текст
  • <i> / <em> — курсив
  • <u> — подчёркнутый текст
  • <code> — моноширинный фрагмент
  • <pre> — блок предварительно форматированного текста
  • <a href="..."> — ссылка
  • <br> — перенос строки

Пример HTML-шаблона:

<b>Новый заказ #{order_id}</b>
Статус: {from_status} → {to_status}
Клиент: {customer_name}
Телефон: {phone}
Email: {email}
Сумма: {total}
Товары:
{items}
<a href="{order_link}">Открыть в админке</a>

MarkdownV2 (Parse Mode)

  • *жирный* — жирный текст
  • _курсив_ — наклонный текст
  • __подчёркнутый__ — подчёркнутый текст
  • ~перечёркнутый~ — перечёркнутый текст
  • ||спойлер|| — скрытый текст
  • [текст](https://example.com) — внешняя ссылка
  • [профиль](tg://user?id=123456789) — ссылка на пользователя
  • `моноширинный` — моноширинный фрагмент
  • ```код``` или ```python код``` — блок кода
  • *_комбинированное форматирование_* — допускаются вложения, главное экранировать специальные символы.

Пример MarkdownV2-шаблона:

*Новый заказ #{order_id}*
Статус: {from_status} → {to_status}
Клиент: {customer_name}
Телефон: {phone}
Email: {email}
Сумма: {total}
Товары:
{items}
[Открыть в админке]({order_link})

Важно: в MarkdownV2 экранируйте специальные символы (например, знак «-», «.», «(», «)» и т. д.), если они встречаются в статичном тексте шаблона.

Безопасность и ограничения

  • Храните токен бота в непубличном месте и не публикуйте его в коде или репозиториях.
  • Telegram ограничивает частоту отправки сообщений: не превышайте ~30 сообщений в секунду для одного бота.
  • Максимальная длина одного сообщения — 4096 символов. Старайтесь сокращать шаблоны или делить большие сообщения.
  • В MarkdownV2 следует экранировать специальные символы (- _ * [ ] ( ) ~ ` > # + = | { } . !), иначе Telegram вернёт ошибку.

Плейсхолдеры

  • {order_id} — номер заказа
  • {from_status} — исходный статус
  • {to_status} — новый статус
  • {customer_name} — имя покупателя
  • {phone} — телефон покупателя
  • {email} — email покупателя
  • {total} — итоговая сумма
  • {items} — список товаров
  • {order_link} — ссылка на заказ
  • {form_title} — название формы Contact Form 7
  • {all_fields} — все поля формы в виде текста
  • {field_{field_name}} — конкретное поле формы (подставьте имя поля вместо field_name)
